The Problem

One app for 500 million people
who have nothing in common.

YONO SBI's biggest drop-off isn't technical — it's relevance. The app shows a retired farmer the same screen as a college student in Pune. Neither stays.

😕 Cognitive overload for new users

A first-time user lands on a 30-feature dashboard with no guidance. Average new-user session: under 90 seconds before drop-off.

🌐 English-first in a Hindi-first country

65%+ of India is not English-primary. Rural and semi-urban users see an interface that doesn't speak their language — literally.

📉 Zero contextual relevance

A student sees home loan ads. A farmer sees investment portfolio trackers. Wrong products shown = zero conversion, zero trust.

🔁 No onboarding intelligence

New users with zero transaction history get the same complex interface as 10-year account holders. Cold-start is completely ignored.

Our Solution

Login → ML detects → Right dashboard loads.

The entire routing happens in under 600ms on-device. No server call. No user input required.

1

Login

User authenticates with username + 6-digit MPIN. 5-attempt lockout security enforced.

2

Signal Extraction

ML pipeline reads 9 features from transaction history — digital spend, pension signals, category diversity, credit score.

3

Segment Prediction

Weighted scoring produces confidence score. ≥68% = auto-route. Below threshold = prediction shown, user confirms.

4

Dashboard Loads

NextGen (English, digital-dense) or Ease (Hindi, simple, scheme-first). User can always switch manually.
